# SHL Assessment Recommendation Agent

A conversational AI agent that recommends SHL assessments based on hiring requirements. The application supports multi-turn conversations, asks clarifying questions when needed, validates recommendations against the SHL assessment catalog, and returns structured JSON responses.

## Features

- Conversational SHL assessment recommendation
- Multi-turn conversation support
- Clarification for vague hiring requirements
- Assessment comparison and refinement
- Catalog-based recommendation validation
- Structured JSON API responses
- FastAPI REST API with automatic Swagger documentation

## Installation

### 1. Clone the repository

```bash
git clone <repository-url>
cd shl-assingement
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Create a `.env` file

Create a `.env` file in the project root.

Example:

```env
LLM_PROVIDER=groq

GROQ_API_KEY=your_groq_api_key
GROQ_MODEL=llama-3.1-70b-versatile
```

## Design Overview

### Architecture

- FastAPI is used to expose REST endpoints.
- The application is stateless; conversation history is provided with each request.
- An LLM generates recommendations using the SHL assessment catalog as context.

### Recommendation Strategy

The complete SHL assessment catalog is supplied to the language model instead of using vector search.

This approach was chosen because:

- The catalog comfortably fits within modern LLM context windows.
- It avoids retrieval errors.
- It simplifies the architecture.
- It reduces infrastructure complexity.

### Validation Pipeline

Every recommendation generated by the model is validated before being returned.

Validation includes:

- Exact catalog matching
- Fuzzy matching fallback
- Duplicate removal
- Maximum recommendation limit enforcement
- URL and assessment type verification
